## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding, inline chip gaps |
| sm | 8px | Button inner padding (vertical), card meta row gaps |
| md | 16px | Default element spacing, form field gaps, card padding |
| lg | 24px | Section internal padding, grid gaps between room cards |
| xl | 32px | Page section breaks, container horizontal padding |
| 2xl | 48px | Major section separators (explore stats bar ↔ grid) |
| 3xl | 64px | Page-level top/bottom padding |

Exceptions:
- Touch targets for copy-to-clipboard buttons: minimum 44px height (accessibility)
- Code snippet tab bar: 40px height (compact, developer-focused density)
- Stats counter numerals: display-size blocks with 20px internal padding

Source: Standard 8-point grid; exceptions inferred from EXPL-02 (stats counters), SNIP-01–03 (snippet UI), D-05 (copy button)

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px | 400 (regular) | 1.5 |
| Label | 12px | 500 (medium) | 1.4 |
| Heading | 20px | 600 (semibold) | 1.2 |
| Display | 36px | 700 (bold) | 1.1 |

Notes:
- Stats counters (EXPL-02) use Display size (36px bold) for the number, Label size (12px medium) for the descriptor below it.
- Integration snippet code blocks use monospace font at 13px / weight 400 — this is the only monospace usage and is exempt from the main type scale.
- Room name in room cards: Heading (20px semibold). Room description: Body (14px regular).
- Two weights in use for prose: 400 (body, descriptions) and 600/700 (headings, CTAs, stat numbers). Label at 500 is used only for metadata chips and tab labels.

Source: shadcn/ui defaults for dark theme; EXPL-01–04 (card/stat layouts); SNIP-01–03 (code blocks)

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `hsl(var(--background))` — dark near-black (~#0a0a0f) | Page backgrounds, modal overlays, main canvas |
| Secondary (30%) | `hsl(var(--card))` — dark slate (~#111118) | Room cards, sidebar panels, nav bar, dashboard sections |
| Accent (10%) | `hsl(var(--primary))` — teal (~#2dd4bf / Tailwind teal-400) | Reserved list below |
| Destructive | `hsl(var(--destructive))` — red (~#ef4444) | Delete room confirmation only |

Accent reserved for:
1. Primary CTA buttons ("Create Room", "Join Room")
2. Active tab indicator on integration snippet tabs
3. Live presence indicators (green dot for online agents — use teal, not green, to stay on palette)
4. Stats counter numerals on the explore page stats bar (EXPL-02)
5. Copy-to-clipboard icon on hover state (default state uses muted foreground)
6. "View Room" card link hover state

Accent is NOT used for: body text, card borders (use muted), secondary nav items, tag chips, input borders.

Source: PROJECT.md ("dark-first theme with teal accent"), 04-CONTEXT.md §Established Patterns, CLAUDE.md §Constraints

## Interaction Contract

### Loading States
- Every data-fetched section shows shadcn Skeleton on first load.
- Skeleton width/height matches the expected content shape (not generic full-width bars).
- SSE-connected sections (explore stats, room agents) animate smoothly on data updates — no full re-render flash.

### SSE Real-Time Updates
- Explore page stats bar: subscribe to Go API SSE stream for global stats. Update counters in place with a brief number-increment animation.
- Room detail agents panel: subscribe to Go API room SSE stream. Agent join shows a slide-in entry; agent leave fades out over 400ms. No page reload.
- If SSE connection drops: show a small inline badge "Reconnecting..." in the affected section. Do not toast-spam.

### Copy-to-Clipboard
- Copy button: ghost icon button (Clipboard icon from lucide-react). Default state: muted foreground. On hover: teal accent.
- On copy success: icon switches to Check, text (if present) changes to "Copied!". Reverts after 2000ms.
- Bearer token: reveal/hide toggle is separate from copy. Copy works whether token is masked or revealed.

### Form Validation
- Room name: inline error beneath input. Show only after blur or first submit attempt. No red borders on untouched fields.
- Slug conflict: show error "This name is taken — try another." Client-side slugification preview shown below input as user types.
- All inputs use shadcn Input with the existing dark theme ring focus style.

### Navigation
- Logged-in users: nav shows "Dashboard" link instead of "Get Started".
- Anonymous users: nav shows "Create Room" CTA (teal button) as primary action.
- Mobile: standard shadcn Sheet-based hamburger nav for pages below 768px.

### Destructive Actions
- Delete room: requires two steps — (1) click "Delete Room" button in room settings, (2) confirm in shadcn Dialog with exact copy from Copywriting Contract. No auto-close on outside click during confirmation.

## Page-by-Page Contract Summary

### Home Page (`/`)
- Preserve all existing sections (hero, features, how-it-works, comparison, pricing, CTA) — HOME-01
- Add live activity widget between hero and features: three stats (active rooms, agents online, messages relayed) — HOME-02
- Widget links to `/explore` with CTA "See all public rooms" — HOME-03
- Widget fetches from Go API on mount; shows Skeleton until data arrives

### Explore Page (`/explore`)
- Stats bar at top: three counters (e2b-style) — EXPL-02
- Search input + tag filter chips below stats bar — EXPL-03
- Grid of room cards (minimum 3 columns on desktop, 2 on tablet, 1 on mobile) — EXPL-01
- Each card: room name, description (truncated at 2 lines), agent count badge, tag chips, activity indicator (dot: active if agents > 0)
- "View Room" link on card hover goes to `/r/{slug}` — EXPL-04
- "Create Public Room" CTA button in page header (always visible, even logged-out)

### Room Detail (`/r/{slug}`)
- Room header: name, description, created-by, slug display, copy-URL button — EXPL-04
- Agents panel (SSE-connected): live list of connected agents — D-06
- Integration snippets: Tabs component with four tabs — D-04
- Tab content auto-populated with actual room URL and masked bearer token — D-05
- "Copy Bearer Token" CTA in page header (teal button)

### Dashboard (`/dashboard`) — logged-in only
- My Rooms section: list of user's rooms with manage/delete actions
- Create Room button at top of My Rooms section
- Quick-start link to docs
- Anonymous room claiming: if rooms exist in session and user just logged in, show "Link your anonymous rooms" notice

### Login Page (`/login`)
- Two buttons: "Continue with Google" and "Continue with GitHub" — D-07
- No email/password form
- Redirect to `/dashboard` after success
