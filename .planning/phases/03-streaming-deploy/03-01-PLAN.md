<objective>
Enable SSE streaming on the A2A relay: configure the a2a-go SDK's built-in SSE transport with heartbeat keep-alive, add the X-Accel-Buffering middleware for Traefik passthrough, implement per-room and server-wide SSE connection limits in the hub, and prove zero goroutine leaks on disconnect with goleak.

Purpose: This is the streaming backbone of Quorum. Without it agents can only use synchronous message/send. With SSE streaming, agents receive real-time event notifications (messages, join/leave) as they happen. The heartbeat keeps connections alive through Traefik. Connection limits protect the VPS from resource exhaustion.

Output: A2A message/stream endpoint functional with 20s heartbeat, SSE proxy header middleware, hub connection limits, goleak disconnect test.

    **2. Create relay/internal/middleware/ssebuffering.go:**

    ```go
    package middleware

    import "net/http"

    // SSENoBuffering sets the X-Accel-Buffering header to "no" on responses
    // whose Content-Type is "text/event-stream". This is CRITICAL for SSE
    // to work through Easypanel's Traefik reverse proxy (per D-04).
    //
    // Without this header, Traefik buffers SSE frames for ~30 seconds and
    // delivers them in a batch, breaking real-time streaming.
    //
    // This middleware should wrap any route group that serves SSE responses,
    // including the A2A JSON-RPC handler (which uses SSE for message/stream).
    func SSENoBuffering(next http.Handler) http.Handler {
        return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
            // Set unconditionally on the route group — only SSE responses
            // will be affected since Traefik only checks this header when
            // Content-Type is text/event-stream. Safe on non-SSE responses.
            w.Header().Set("X-Accel-Buffering", "no")
            next.ServeHTTP(w, r)
        })
    }
    ```

    **3. Mount the middleware on the A2A route group:**

    In relay/internal/relay/handler.go (or relay/cmd/server/main.go — wherever the A2A subrouter is defined), wrap the A2A handler path with the SSENoBuffering middleware:

    ```go
    // Where the A2A routes are mounted on chi:
    r.Route("/r/{slug}/a2a", func(r chi.Router) {
        r.Use(middleware.SSENoBuffering) // Must be before the JSON-RPC handler
        r.Handle("/*", jsonrpcHandler)
    })
    ```

    If the A2A routes are mounted differently (e.g., via `r.Mount`), apply the middleware in the same pattern — it must wrap the handler that serves SSE responses.
